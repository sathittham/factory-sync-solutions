## 1. Test Scope

### 1.1 In Scope
- **Unit tests** (Vitest + jsdom + Testing Library): `apps/web-app/src/**/*.test.{ts,tsx}` (39 files)
  - Route guards: `AuthGuard`, `AdminGuard`, `CompanySettingsGuard`, `RegisterGuard`
  - Redux store: `authSlice`, `quizSlice`, `store/index`
  - Library modules: `api`, `firebase`, `queries`, `queryClient`, `analytics`, `dayjs`, `i18n`, `theme`, `utils`
  - Hooks: `useAuth`, `use-mobile`
  - Pages: `SignInPage`, `RegisterPage`, `DashboardPage`, `QuizPage`, `ResultPage`, `ProfilePage`, `CompanySettingsPage`, `AdminPage`, `AuthActionPage`, `NotFoundPage`
  - Feature components: `Layout`, `PageLayout`, `AuthPanel`, `LegalModal`, `ProfileDialog`, `Turnstile`, `login-form`, `CookieConsent`, `motion`, `form/select-field`
- **End-to-end tests** (Playwright): `apps/web-app/e2e/*.spec.ts` (7 files)
  - Smoke: sign-in form, login → dashboard, one guarded-route redirect, 404
  - Regression: login, navigation/guards, theme persistence, cookie consent + Consent Mode v2, auth-action (`/auth/action` oobCode flows), accessibility (a11y)

### 1.2 Out of Scope
- Backend API, Firestore, Firebase Admin — validated in `apps/backend`.
- Third-party SDK internals (Firebase client SDK, Cloudflare Turnstile widget) — mocked at the boundary, not exercised for real.
- shadcn/ui primitives under `src/components/ui/**` — excluded from the coverage gate (thin Radix wrappers, low business-logic value); covered indirectly through the feature components that use them.
- Thin composition/wiring files — `App.tsx`, `router.tsx`, `routeTree.gen.ts` (auto-generated by the TanStack Router Vite plugin), `src/routes/**` (one-line `createFileRoute(...)` declarations) — excluded from the coverage gate as declarative wiring, not logic.
- Visual-regression and automated accessibility (axe) audits beyond what `a11y.spec.ts` already checks (title, alt text, keyboard access, ARIA roles, base font size).
- Superadmin/staff-role-specific backoffice concerns — out of scope for this app entirely (see `web-backoffice`'s own test plan).

### 1.3 Test Environment
| Environment | Details |
|---|---|
| Unit | `pnpm --filter @repo/web-app test` (Vitest + jsdom). Global setup in `src/test/setup.ts` (`@testing-library/jest-dom`, `__APP_VERSION__` stub). |
| Coverage | `pnpm --filter @repo/web-app test:coverage` (v8, `all: true` over `src/**`, hard-enforced 70% gate on statements/branches/functions/lines). |
| E2E (local) | `pnpm --filter @repo/web-app test:e2e` — auto-starts a local Vite dev server on `localhost:5173`. Requires `e2e/.env.e2e.local` (copy from `.env.e2e.example`) with `E2E_USER_EMAIL`/`E2E_USER_PASSWORD` for a registered test account. |
| E2E (deployed) | `E2E_BASE_URL=https://app-staging.factorysyncsolutions.com playwright test` — no local server. |
| E2E subsets | `pnpm test:e2e:smoke` (`--grep @smoke`, fast critical-path net) · `pnpm test:e2e:regression` (`--grep @regression --grep-invert @flaky`, full flow coverage minus quarantined tests). |
| Browsers | Local: chromium, firefox, webkit, mobile-chrome. |

## 6. Known Gaps & Follow-ups

| Gap | Priority | Note |
|---|---|---|
| `ProfileDialog` Save button doesn't reactively re-enable after a field edit | Medium | Reads `form.state.isDirty`/`isSubmitting` directly in JSX instead of via `form.Subscribe`/`useStore` — editing a field updates the TanStack Form store without re-rendering `ProfileDialog`. Documented via a regression test; needs a `frontend-dev` fix wrapping the disabled expression in `form.Subscribe`. |
| `CompanySettingsPage` success banner never visibly renders | Low | A `useEffect` keyed on `[profile, form]` resets `success` back to `false` in the same commit that a successful save sets it `true` (the Redux `setProfile` dispatch re-triggers the effect). Documented in `CompanySettingsPage.test.tsx`; needs a `frontend-dev` follow-up to gate the reset on an actual profile *change*, not any profile reference update. |
| `ProfilePage.tsx` / `ProfileDialog.tsx` branch coverage still below file average (~49-51%) | Low | Global gate passes without them; lowest-hanging branches were prioritized first. Room for deeper edge-case tests in a future pass. |
| Turnstile real-widget verification | Low | Mocked at `globalThis.turnstile` in unit tests; no e2e test exercises the real Cloudflare widget (needs Turnstile test keys). |
| Accessibility (axe) + visual regression beyond `a11y.spec.ts` | Low | Not yet adopted repo-wide (same gap noted in `web-official`'s test plan). |
